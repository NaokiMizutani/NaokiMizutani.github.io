R, RStudio, Shiny のインストール

June 8, 2016

CentOS7 への R, RStudio, Shiny のインストールについてのメモ

R のインストール

【方法１】

基本的に yum でインストールできるようだが、 epel のレポジトリを登録する必要がある。

# yum install epel-release
# yum install R

【方法２】

CRANサイト のトップページに、 各OSへのダウンロードに関する説明へのリンクが貼られており、 Linux/redhat 用の Readme ファイルによると、以下のようにインストールを進めるとのことである。

EPELサイト で newest ‘epel-release’ package for EL7 を 確認する。

その URI を用いて、

# rpm -ihv https://dl.fedoraproject.org/pub/epel/epel-release-latest-7.noarch.rpm
# yum install R

を実行する。

RStudio のインストール

RStudioサイト のページで、 RStudio Desktop の Open Source Edition のダウンロードボタンを押し、 CentOS7 へのインストール用として、 Fedora/Redhat 用 64bit インストーラ rpmパッケージが あることを確認する。

その URI を用いて、 # rpm -ivh https://download1.rstudio.org/rstudio-0.99.902-x86_64.rpm でインストールできる。

なお、 * rpm パッケージのインストールは、ルート権限で、

`# rpm -ivh package.rpm`

  • インストールしたい RPM パッケージの古いバージョンが既にインストールされている場合は

    # rpm -Uvh package.rpm

  • ある RPM パッケージが既にインストールされているかどうかを調べる場合は

    # rpm -q package

を使う。

起動は、RStudio のアプリケーションアイコンが「開発」のところに入るので、そこから起動する。

Shiny のインストール

R を root で起動し、

> install.packages('shiny', repos='https://cran.rstudio.com/')

その他の方法として、RStudio の右下のサブウィンドウの ‘Packages’ タブで ‘Install’ ボタンを 押すことによって、ダイアログが開き、Packages 欄に ‘shiny’ と入力し、’Install dependencies’ に チェックを入れると、依存関係にある必要なライブラリも導入される。
